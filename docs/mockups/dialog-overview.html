1 · Termin stornieren

AppointmentCancelConfirmDialog AlertDialog roh ⚠
Termin stornieren?

Der Termin wird dauerhaft als storniert markiert. Alle Mitarbeiter werden vom Termin abgezogen und sind im Terminzeitraum zur erneuten Planung verfügbar. Der Auftragswert wird im System auf 0,- Euro gesetzt. Stornierte Termine können nicht reaktiviert werden.

Inkonsistenzen:
• Kein Header-Hintergrund (bg-slate-50) — alle anderen Dialoge haben ihn
• Kein Icon-Frame links im Header
• Kein X-Schließen-Button oben rechts (AlertDialog hat keinen)
• Kein border-b unter dem Header
• Verwendet AlertDialogContent ohne gap-0 overflow-hidden p-0 Reset — hat noch Padding-Innenraum
• Zerstörender Button ohne border border-destructive-border Klasse (nur inline style)

2 · Wochenplanung blockieren (ConfirmDialogBase)

ConfirmDialogBase via TourManagement AlertDialog (ConfirmDialogBase)
🗺
Wochenplanung blockieren?

KW 23 / 2026 wird blockiert. Danach sind Ressourcenänderungen in dieser Tour-KW gesperrt, bis sie wieder freigegeben wird.

Hinweis: ConfirmDialogBase korrekt umgesetzt — Icon-Frame, Header bg-slate-50, Border, Footer bg-slate-50 vorhanden.
• Kein X-Button (AlertDialog-verhalten korrekt für Bestätigungs-Dialoge)
