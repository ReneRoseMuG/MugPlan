# Journalpflege

## Regeln

- Neue Journaleinträge werden als Markdown-Dateien unter `docs/wiki/journal/` gepflegt.
- Neue Einträge werden oben unter `## Einträge` in `docs/wiki/journal/README.md` verlinkt.
- Umfangreiche Aufgaben, Refactorings und mehrschichtige Implementierungen erhalten einen eigenen Journaleintrag.
- Sichtbare Datumsangaben verwenden das Kurzformat `dd.MM.yy`.
- Ab 07.05.26 ersetzt das Wiki-Journal neue Auftragslogs. Der Ordner `logs/` bleibt historischer Altbestand.

## Aufgabenbezug

- Wenn ein Journaleintrag eine Aufgabe abschließt, wird die Aufgabe im Feld `Journal` auf den Eintrag verlinkt.
- Abgeschlossene oder verworfene Aufgaben werden nach `docs/wiki/tasks/closed/` verschoben.
- Offene Aufgaben bleiben in `docs/wiki/tasks/README.md` sichtbar.
